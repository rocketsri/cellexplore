# v7 — The amortized-selection resolution, and non-localization as a theorem

Two results this round:

- **The amortized-selection resolution of v6's "search-limited" verdict.** v6 asked the wrong low-`I`
  question ("can ES shape the whole ~1800-dim body"). B1/O2 say the *recurring* cost is only the
  per-target code (`log₂K` bits); the body is a **one-time, amortized** cost — biology's conserved
  developmental toolkit, evolved once over deep time, not re-paid per organism. The right question is
  therefore: **given the toolkit, can a purely scalar, non-differentiable, per-outcome shaper *select* a
  held-out target via its small code?** v7 answers **yes** (§2) — so the per-target generativity
  `D̄ − log₂K` is delivered by a genome-like process, with only the toolkit expensive, exactly as in
  biology.
- **B3 — non-localization + self-repair + the H4 double edge, as one theorem** (§3), replacing v5's
  leaky re-coarsening test with an exact code-distance bound.

---

## 1. The amortized split (why v6's negative was the wrong question)

v6 established: (i) a target-agnostic shared body — a generic morphogenetic prior — **exists** (backprop
reaches held-out corr 0.78 via a 12-param code); (ii) low-`I` ES cannot shape that whole body. But B1
and O2 locate the *recurring* generativity in the **per-target code**, not the body:

| cost | what it is | when paid | biology | charged per target? |
|---|---|---|---|---|
| **toolkit** `|θ|` | the shared FiLM-NCA rule (the repertoire/codebook) | **once** | conserved developmental toolkit, evolved over deep time | **no** (amortized over all targets & all time) |
| **selection** `log₂K = |code|` | which attractor of the repertoire | **per target** | regulatory tweak per lineage | **yes** |

So the correct low-`I` test is on the *selection* step: on a **frozen** toolkit, can a purely scalar,
non-differentiable, per-outcome shaper find the small code for a **held-out** target? If yes, the
per-target generativity is genome-like even though the one-time toolkit is expensive — which is precisely
biology's arrangement (a species pays for its body plan once, over deep time; each individual's
morphology is then a cheap regulatory setting on that shared plan).

## 2. Experiment v7 — scalar-ES selection on a frozen toolkit

`experiments/v7_amortized_selection.py`: (1) train the toolkit **once** by backprop on `K_train=24`
targets (the amortized deep-time analog — expensive, paid once); (2) **freeze** it; (3) for **held-out**
targets, find the 12-param code by **pure scalar ES** (antithetic, rank-based; the only signal is a
scalar correlation per rollout — non-differentiable, per-outcome, genome-like); compare to backprop
code-fit and to scalar-ES on a random body; (4) sweep code dim `d`.

Executed (`experiments/v7_results.log`, ~4 min):

| selection method (on the **same** frozen toolkit) | mean held-out corr |
|---|---|
| **scalar ES** (non-differentiable, per-outcome, 5600 rollouts) | **0.375** |
| backprop (gradient through the code) | 0.410 |
| scalar ES on a **random** body (control) | **0.079** |

**The finding:** on the same frozen toolkit, a purely scalar, non-differentiable shaper selects held-out
targets **as well as gradient** (0.375 vs 0.410), and both are far above the random-body control (0.079).
Per-target **selection is low-`I`-discoverable** — indeed on one held-out target scalar ES (0.43) beat a
stuck backprop run (−0.01), the small code landscape being benign for gradient-free search. Because the
comparison is on one shared body, it is **toolkit-invariant**: this run's toolkit trained to only 0.53
(vs 0.78 in v6's existence run — backprop has run-to-run variance), which lowers all absolute held-out
numbers, but scalar-ES ≈ backprop ≫ control holds regardless. The recurring per-target generativity
`Δ = D̄ − |code| = 400 − 72 = +328 bits` is therefore delivered by a **genome-like** (per-outcome,
non-differentiable, low-bits-per-query) selection process; only the one-time toolkit is expensive.

Code-dimension sweep (scalar ES, a fresh toolkit retrained per `d` — so noisy): held-out corr
`d=2 → 0.29`, `d=4 → 0.40`, `d=8 → 0.25`, `d=12 → 0.51`; all ≫ the 0.079 control at every `d`. Selection
succeeds across code widths; the trend is dominated by per-`d` toolkit-retraining variance, not by `d`.

*Honest caveat:* absolute selection fidelity (~0.37) is modest, not a crisp reconstruction — it reflects
the weak toolkit and the 8×8 scale. The claim is the **comparison and the mechanism** (scalar low-`I`
selection ≈ gradient, ≫ control), not high-fidelity image recovery.

## 3. B3 — non-localization, self-repair, and the H4 double edge as one theorem

v5's re-coarsening test was leaky: a 1-cell fragment regrew the target (the *shared rule* did the work),
so it did not actually test "no `o(N)` subset **holds** the target." B3 fixes this by distinguishing
**holding** (decoding the target from a subset's *state*) from **regenerating** (the shared rule
recomputing it), and giving exact bounds via coding theory.

**Model.** Represent the maintained target as a codeword `x ∈ Σ^N` of an `[N, k, Δ]` code over the
per-cell alphabet `Σ` (the visible morphology is a fixed function of `x`). "A subset `S` of cells holds
the target" means the target is determined by (decodable from) the states `x_S` alone.

> **Theorem B3.**
> **(a) Non-localization.** For an MDS code (e.g. Reed–Solomon; or a random code w.h.p.), any subset of
> `s < k` cells leaves the codeword — hence the target — information-theoretically *undetermined*:
> `≥ |Σ|^{k−s}` codewords are consistent with `x_S`, so `I(x_S ; target) < H(target)` and no decoder
> recovers it. With a **constant-rate** code `k = ρN`, **no `o(N)` subset holds the target**; the coarsest
> subset at which "no subset holds it" first fails is exactly `s = k = Θ(N)`. This is guard 2 as a
> theorem, with the exact threshold `k` — and it strictly sharpens v5's counting bound
> `s ≥ D_ε/(C·q)` (which only says the bits cannot *fit*; B3(a) says fewer than `k` are provably
> *ambiguous* even if they fit).
> **(b) Self-repair with a proven distance.** Any erasure of `e ≤ Δ − 1` cells is exactly correctable:
> the target is recovered from the survivors (for MDS, any `k` surviving cells suffice). Regeneration
> tolerance is `Δ − 1` erasures — *proven*, unlike the NCA's empirical recovery curve.
> **(c) The H4 double edge, quantified.** Changing the maintained target requires altering `≥ Δ` cells:
> any edit of `< Δ` cells is reverted by error-correction. Hence **erasure-tolerance and
> resistance-to-correction are the same `Δ`** — you cannot buy self-repair radius `Δ/2` without paying
> correction-cost `Δ`. H4 is thus not a qualitative caveat but a conservation law: robustness radius +
> correctability are traded off through a single code parameter.

**Proof.** (a) An `[N,k,Δ]` MDS code has `Δ = N−k+1`; any `s < k` coordinates extend to `|Σ|^{k−s}`
distinct codewords (the code is a dimension-`k` subspace; a projection onto `s<k` coordinates has kernel
of dimension `≥ k−s`), so the target is not a function of `x_S`. (b) Erasure decoding of MDS codes
corrects any `N−k = Δ−1` erasures. (c) Two codewords differ in `≥ Δ` positions (minimum distance), so
moving from the maintained codeword to any *other* valid target requires `≥ Δ` cell edits; fewer than
`Δ/2` edits are corrected back by nearest-codeword decoding. ∎

**Honesty / positioning.** The NCA does not literally implement an MDS code; B3 is the *idealized limit*
showing the three properties (non-localization with threshold `k`, self-repair with radius `Δ−1`, and
the H4 tradeoff) are jointly, rigorously achievable, and it gives the exact object a construction should
approximate. The v5 NCA is an *approximate, imperfect* instance: its regeneration (0.79) and its counting
bound (17% of `N`) are the empirical shadows of B3(b) and B3(a). The gap between the NCA's soft
distributed code and an MDS code is the exact residual — a construction that maintains a
provable-distance code as its attractor would upgrade v5's empirical self-repair to B3's bounds.

## 4. Standing after v7

The generativity question is now fully triangulated:

- **O2 (v5):** necessary shaping is `log₂K` (selection), not `D` (specification).
- **B1 (v6):** `log₂K` is sufficient **iff** the shared rule is a target-agnostic codebook (held-out
  realizability); achievable marginal generativity `= D̄ − log₂K`.
- **v6 existence + v7 selection:** such a codebook **exists** (backprop), and — the new result — the
  **per-target selection on it is low-`I`-discoverable** by a purely scalar, non-differentiable shaper
  (§2). The only expensive part is the **one-time** toolkit, which is amortized (biology pays it over
  deep time).
- **B3 (v7):** non-localization, self-repair, and the H4 double edge are one code-distance theorem, with
  exact thresholds.

**Net:** the ambitious problem's *recurring* content — generate more target-specification per target than
the per-target shaping supplies, as a self-repairing non-local attractor — is **achieved in the
amortized/marginal sense by a genome-like (per-outcome, non-differentiable, low-bits-per-query) process**,
with the one expensive ingredient (the shared toolkit) being exactly the one biology amortizes over
evolutionary time. The remaining honest gap is the **one-time toolkit's** own shaping: whether a *low-`I`*
(not backprop) process can build the generic codebook from scratch at feasible query cost — the deep-time
question — plus H3 (Type-B defection), still untouched.
