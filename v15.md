# v15 — Associative memory: retrieval (Claim-A), not a new generativity substrate

Adjudicates the 4th new-agenda crux (compute-light, `experiments/v15_associative_memory.py`): is a
learned associative memory (modern Hopfield / dense associative memory = attention) a **generativity**
mechanism (realizes held-out targets via a key) or mere **retrieval** (Claim-A: the target was stored,
so the selection specifies it)?

Modern Hopfield retrieval `x*(q) = Ξ softmax(β Ξᵀq)` always outputs a (softmax-weighted) combination of
**stored** patterns. Measured (`n=64`, `K=40` stored):

| test | corr | reading |
|---|---|---|
| **retrieval** (noisy query of a stored pattern) | **0.912** | works as a memory |
| **held-out UNRELATED** (new random target, optimize key) | **0.476** | out-of-span component is unreachable → **RETRIEVAL = Claim-A** |
| **held-out COMPOSITIONAL** (target in span of stored atoms) | 0.692 | reachable only as a *constrained* combination |

**Verdict.** Plain associative memory is **Claim-A (retrieval)**: it realizes stored patterns and, weakly,
in-span combinations, but **not** out-of-span held-out targets (0.48 vs 0.91 for stored). Where it *does*
generalize — compositional, in-span targets — it **is the superposition/dictionary substrate (v14)** with
the stored patterns as atoms and the softmax key as a (simplex-constrained) code; and it is *more
constrained* than free sparse coding (softmax weights live on a probability simplex), so it generalizes
*worse* (0.69 vs v14's 0.98). **Registry result: associative memory is not a distinct new positive
substrate** — it is either Claim-A (retrieval) or a restricted case of superposition.

**Why this is useful.** It sharpens the discriminator once more: "the selection retrieves a stored
target" (high fidelity, but the target was *supplied* = Claim-A) is exactly what held-out realizability
is designed to catch, and it does — associative memory scores high on stored recall and low on held-out,
the Claim-A fingerprint. The two genuine positives (NCA, superposition/dictionary) both *generalize* to
held-out; associative memory does not, which is why it is retrieval, not generativity.
