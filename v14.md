# v14 — A second positive generativity substrate: superposition / sparse dictionaries (→ SAEs)

New-agenda round (parallel-agent exploration beyond the retracted Holonomy line). This is the substrate
that connects the program's positive result to **real trained networks**: superposition / sparse
overcomplete codes / SAEs. Executed: `experiments/v14_superposition_codebook.py` (pure numpy).

## The construction (B-bio on a superposition substrate)

- **Shared prior / codebook**: an overcomplete dictionary `Φ` (`n=48` units, `M=96` atoms). Amortized,
  target-agnostic.
- **Per-target selection**: a `k=5`-sparse code `s`, `x ≈ Φs`. Selection cost `≈ k·log₂(M/k) ≈ 53` bits.
- **Attractor dynamics**: sparse coding itself (OMP / ISTA / LCA is a recurrent local dynamics whose
  fixed point is the code) — a distributed representation where no single neuron holds a clean feature
  (superposition).
- **Generativity = held-out realizability**: a dictionary shaped on `K_train` targets realizes
  **held-out** targets (never trained on) via a fresh sparse code — the exact B1 test, on this substrate.

## The control was load-bearing (and first exposed a fake result)

The lost first attempt used **soft/overcomplete** sparse coding, where a *random* dictionary reconstructs
held-out targets as well as a learned one (0.964 vs 0.964) — because an overcomplete random dictionary
already spans the space. That "generativity" was fake (the same failure the audit taught us to catch).
The fix is **hard `k`-sparse coding (OMP)**: only a dictionary that *contains the right atoms* can
reconstruct a `k`-sparse signal with `k` atoms; a random dictionary cannot cheat. Random-dict control
then drops to **0.690**.

## Result (executed, with the honest control)

| | held-out recon corr | atom-recovery | dim_sel |
|---|---|---|---|
| control (random dictionary) | **0.690** | — | — |
| learned, `K_train=50` | 0.767 | 0.01 | 11.1 |
| learned, `K_train=200` | 0.853 | 0.10 | 13.7 |
| learned, `K_train=600` | **0.978** | **0.885** | 15.4 |

- **Held-out realizability 0.978 ≫ 0.690 control** — a genuine second positive substrate. The learned
  dictionary **recovers 88% of the true generating atoms** (`K_train=600`), so it is not cheating; it
  recovered the structure.
- **B1 sufficiency signature IS seen here** (held-out rises 0.77 → 0.85 → 0.98 with `K_train`) — unlike
  the NCA (v12), where held-out was already saturated at small `K_train`. Dictionary identifiability
  needs `K_train ≫ M`, so the rise is expected and clean.
- **Low-`I` arm**: a non-differentiable, per-outcome ES (scalar reconstruction-corr fitness) in a
  low-dim random reparametrization of dictionary space also generalizes (held-out 0.98 > 0.69 control) —
  low-`I` shaping builds the dictionary too. *(Honest caveat: this ES number is not a clean
  backprop-beating claim; it is above control, which is the load-bearing comparison.)*
- **Generativity** `Δ = D̄ − |code| ≈ 192 − 53 = +139` bits/target, measure-consistent (both target and
  code are coded under the *same* dictionary decoder; the dictionary is a proven target-agnostic codebook
  because it realizes held-out targets).

## Honest scope

- **Non-localization is weaker here than the program's guard-2 ideal.** Superposition distributes each
  feature across neurons (no single neuron holds a clean feature — real, and what trained nets exhibit),
  but the *code* of a given target lives in its `k` active atoms, a small subset. So this substrate is
  strong on **generativity** (the prize) and on *neuron-level* non-localization, but does not give the
  *loop-integral* "no o(N) subset at all" that the (retracted-for-generativity) Holonomy encoding did.
- **Ground-truth-sparse targets.** The clean result uses targets that are genuinely `k`-sparse in a true
  dictionary `D0`; the honest generalization claim is "a learned dictionary recovers `D0` and sparsely
  codes held-out draws." Real SAEs demonstrably generalize to held-out activations — this is the toy of
  that phenomenon, with the controls made explicit.

## Why this matters for the program

It is the **second** substrate (after the NCA) where a **shared learned prior + cheap per-target
selection realizes held-out rich targets** — the amortized-codebook picture (O2 + B1) — and it is the one
that maps directly onto **mainstream ML**: a trained model's superposed features are a target-agnostic
codebook; a prompt/context selects a sparse combination realizing a specific rich behavior; held-out
realizability is exactly why the features generalize. The program's positive result is thus not an
NCA-specific curiosity — it recurs on the substrate real networks actually use.
